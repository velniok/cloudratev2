# CloudRate v2

> Веб-приложение для оценки музыки российских андерграунд-исполнителей с платформы SoundCloud.

🔗 **[Live Demo](https://cloudratev2.vercel.app/)**

---

## О проекте

CloudRate — это платформа, где пользователи могут открывать для себя музыку российского андерграунда, оценивать треки и оставлять отзывы. Это вторая версия приложения, переписанная с нуля на современном стеке с переработанной архитектурой и структурой проекта.

Проект создан в учебных целях и является частью моего портфолио. Основная цель — закрепить навыки fullstack-разработки: проектирование REST API, работа с базой данных, построение масштабируемой фронтенд-архитектуры и деплой приложения в продакшн.

---

## Стек технологий

### Frontend
- **React + Vite** + **TypeScript**
- **Redux Toolkit** — глобальное управление состоянием
- **Axios** — HTTP-запросы
- **SCSS Modules** — изолированная стилизация компонентов
- **Feature-Sliced Design (FSD)** — архитектурная методология

### Backend
- **Node.js** + **Express** + **JavaScript**
- **PostgreSQL** — реляционная база данных
- **Cloudinary** — облачное хранилище для изображений
- **bcrypt** — хэширование паролей
- **JWT** — аутентификация на основе
- **cookie-parser** - хранение куков
- **Nodemailer** - отправка кода подтверждения

### Деплой
- **Vercel** — деплой клиента и сервера

## Функциональность

### ✅ Реализовано

- Регистрация с подтверждением email по 6-значному коду
- Авторизация (JWT + bcrypt) с access-токеном (30 мин) и refresh-токеном (30 дней) в httpOnly cookie
- Профили пользователей с возможностью редактирования и загрузкой аватара (Cloudinary)
- Страницы артистов и треков с подробной информацией
- Оценка треков с необязательным текстовым отзывом
- Возможность ставить лайки отзывам и подписываться на артистов
- Страница поиска с фильтрами по трекам, артистам и пользователям
- Toast-уведомления (UX) для обратной связи с пользователем (успех, предупреждение, ошибка)
- Адаптация для мобильных устройств
- Админ-панель:
  - Управление треками, артистами и пользователями (CRUD)
  - Полный список треков, артистов и пользователей с пагинацией
  - Просмотр общей статистики приложения

### 🚧 В разработке

- Доработка главной страницы: тематические блоки с рекомендациями — новые отзывы, топ по оценкам и т.д.
- Отправка заявки артиста на его верификацию
- Отправка заявок на добавление новых треков от пользователей
- Подсветка отзыва пользователя на трек, если на него поставил лайк исполнитель трека

## Архитектура

Фронтенд построен по методологии **Feature-Sliced Design**, что обеспечивает чёткое разделение ответственности, масштабируемость и удобство поддержки кода.

```
client/
├── app/          # инициализация приложения, провайдеры, роутинг
├── pages/        # страницы (Home, Artist, Track, Profile, Admin...)
├── widgets/      # самостоятельные блоки интерфейса (шапка, карточки...)
├── features/     # пользовательские сценарии (авторизация, оценка трека...)
├── entities/     # бизнес-сущности (User, Track, Artist)
└── shared/       # UI-кит, утилиты, api, константы
```

Бэкенд построен на **Express** с разделением на роуты, контроллеры и middleware. База данных — **PostgreSQL**, взаимодействие через raw SQL-запросы.

```
server/
├── src/
│   ├── config/       # конфигурация БД и сторонних сервисов
│   ├── controllers/  # логика обработки запросов
│   ├── middlewares/  # JWT-аутентификация, обработка ошибок, проверка ролей
│   ├── routes/       # роуты API (users, tracks, artists, auth...)
│   ├── services/     # бизнес-логика, работа с БД
│   ├── utils/        # вспомогательные функции
│   └── app.js        # точка входа
```

## Безопасность

- Access-токен хранится в Redux (память), живёт 30 минут
- Refresh-токен хранится в httpOnly cookie (недоступен для JavaScript), живёт 30 дней
- При истечении access-токена клиент автоматически обновляет его через /refresh, используя refresh cookie
- После перезагрузки страницы сессия восстанавливается вызовом /refresh
- Все запросы к API идут через axios-интерцепторы, добавляющие access-токен в заголовок Authorization
- Защита от CSRF обеспечена флагами SameSite=Secure на refresh cookie

## Чем v2 отличается от v1

Первая версия CloudRate была учебным проектом без чёткой структуры — весь код лежал в одной куче, стили были глобальными и конфликтовали между собой, TypeScript отсутствовал. В какой-то момент поддерживать и развивать это стало сложнее, чем переписать заново.

В v2 были переосмыслены все подходы:

|  | v1 | v2 |
|---|---|---|
| Архитектура | Отсутствует | Feature-Sliced Design |
| Типизация | JavaScript | TypeScript (frontend) |
| Стили | Глобальный SCSS | SCSS Modules |
| Дизайн | Самописный, непоследовательный | Переработан с нуля |
| Деплой | Отсутствует | Vercel (client + server) |
